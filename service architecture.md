# 덕앤플라워 홈페이지 개발 아키텍처 문서

## 1. 문서 개요

| 구분 | 내용 |
|---|---|
| 문서명 | 덕앤플라워 홈페이지 개발 아키텍처 문서 |
| 대상 서비스 | 덕앤플라워 농원 홈페이지 |
| 서비스 목적 | 농원 정보 제공을 통한 잠재 고객 방문 유도 |

| 확장 가능성 확보 | 추후 예약, 상품관리, 관리자 기능 추가 가능하도록 설계 |

## 4. 전체 아키텍처 구조

```text
사용자
  ↓
덕앤플라워 홈페이지 접속
  ↓
Vercel CDN / Hosting
  ↓
Next.js 정적 페이지
  ↓
Markdown 콘텐츠 + 이미지 파일
  ↓
전화 / 카카오톡 / 지도 연결
```

| 구성 요소 | 역할 |
|---|---|
| 사용자 | 모바일 또는 PC로 홈페이지 접속 |
| Vercel | 홈페이지를 인터넷에 배포하고 빠르게 제공 |
| Next.js | 페이지 구성, 라우팅, SEO 설정 처리 |
| Markdown/MDX | 농원 소개, 식물 정보, 관리법 글 저장 |
| 이미지 폴더 | 농원 사진, 화분 사진, 묘목 사진 저장 |
| 외부 링크 | 전화, 카카오톡, 네이버지도, 카카오맵 연결 |

---

## 5. Vercel을 선택하는 이유

| 이유 | 설명 |
|---|---|
| 정보형 홈페이지에 적합 | 예약/구매 기능이 없는 초기 홈페이지에 충분 |
| Next.js와 궁합이 좋음 | Next.js 프로젝트 배포가 매우 간단 |
| 무료 또는 저비용 시작 가능 | 초기 트래픽이 많지 않은 농원 홈페이지에 적합 |
| GitHub 자동 배포 가능 | 코드 수정 후 GitHub에 올리면 자동 배포 가능 |
| 속도와 안정성 우수 | CDN 기반으로 모바일 접속 속도에 유리 |
| 검색 노출 대응 가능 | 페이지별 메타태그, 사이트맵, 구조화된 콘텐츠 구성 가능 |
| 확장성 보유 | 추후 DB, 예약, 관리자 기능을 붙일 수 있음 |

## 6. 초기 제외 기능

| 기능 | 초기 제외 이유 |
|---|---|
| 방문 예약 기능 | 현재 목표가 정보 제공 중심이므로 불필요 |
| 온라인 구매 기능 | 결제, 재고, 배송 관리 부담이 큼 |
| 회원가입 | 고객 정보 저장 목적이 아직 없음 |
| 장바구니 | 온라인 판매 기능과 함께 추후 검토 |
| 관리자 페이지 | 초기에는 Markdown과 이미지 파일 수정으로 충분 |
| 데이터베이스 | 저장해야 할 동적 데이터가 아직 없음 |
| 백엔드 API 서버 | 페이지 제공과 외부 링크 연결만으로 충분 |

## 10. Markdown 콘텐츠 예시

```markdown
---
title: "블루베리 묘목"
category: "유실수"
location: "마당, 텃밭, 전원주택"
difficulty: "보통"
season: "봄, 가을"
image: "/images/plants/blueberry.jpg"
---

블루베리는 전원주택 마당이나 텃밭에 심기 좋은 유실수입니다.

햇빛이 잘 드는 곳에서 잘 자라며, 산성 토양을 좋아합니다.

덕앤플라워에서는 초보자도 이해하기 쉬운 묘목 정보를 제공합니다.
```

| 장점 | 설명 |
|---|---|
| 수정이 쉬움 | 글과 사진 경로만 바꾸면 콘텐츠 수정 가능 |
| DB가 필요 없음 | 초기 정보형 홈페이지에 적합 |
| 검색 노출에 유리 | 페이지별 제목과 설명을 명확하게 관리 가능 |
| 확장 가능 | 나중에 CMS로 전환하기 쉬움 |

## 15. 이미지 관리 기준

| 이미지 종류 | 권장 기준 |
|---|---|
| 파일 형식 | JPG 또는 WebP |
| 파일명 | 영문 소문자와 하이픈 사용 |

| 대표 이미지 | 밝고 선명한 실제 농원 사진 사용 |
| 상품 이미지 | 같은 비율로 촬영하여 카드형 UI에 적합하게 관리 |
| alt 텍스트 | “덕앤플라워 영천 농원 밭 전경”처럼 설명형으로 작성 |

## 17. Vercel 배포 구조

```text
개발자 PC
  ↓ git push
GitHub 저장소
  ↓ 자동 연결
Vercel
  ↓ 자동 빌드
덕앤플라워 홈페이지 공개
```

| 장점 | 설명 |
|---|---|
| 자동 배포 | GitHub에 수정사항을 올리면 Vercel이 자동 반영 |
| 미리보기 가능 | 수정 작업마다 Preview URL 생성 가능 |
| 운영 편의성 | 별도 서버 관리가 필요 없음 |
| 속도 | CDN 기반으로 빠르게 제공 |

## 20. 향후 확장 아키텍처

| 확장 기능 | 도입 시점 | 추천 방식 |
|---|---|---|

| 블로그 운영 | 검색 유입을 늘릴 때 | Markdown 기반 글쓰기 또는 CMS 연동 |
