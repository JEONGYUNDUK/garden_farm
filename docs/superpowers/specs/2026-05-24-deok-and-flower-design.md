# 덕앤플라워 홈페이지 디자인 스펙

## 1. 목표

덕앤플라워 홈페이지는 경북 영천의 농원 정보를 신뢰감 있게 전달하고, 사용자가 농원을 직접 방문해 보고 싶다는 인상을 갖게 만드는 정보 제공형 브랜드 사이트로 구축한다. 초기 버전은 예약, 구매, 회원가입 없이 빠르게 공개 가능한 정적 사이트를 목표로 하며, 로컬 작업 후 GitHub에 푸시하고 Vercel로 배포하는 운영 방식을 전제로 한다.

## 2. 제품 방향

- 사이트 형태는 다중 페이지가 아닌 단일 페이지 랜딩 구조로 한다.
- 상단 네비게이션은 각 섹션으로 이동하는 앵커 링크 방식으로 구성한다.
- 화면 분위기는 깔끔한 브랜드형을 기본으로 하되, 농원의 자연스러움은 색상과 이미지 톤으로 보완한다.
- 반복 콘텐츠는 컴포넌트 내부 하드코딩 대신 별도 데이터 파일로 분리해 유지보수를 쉽게 한다.
- 초기 공개 후 실제 농원 사진, 소식, 식물 정보가 늘어나더라도 구조 변경 없이 데이터만 추가할 수 있도록 설계한다.

## 3. 기술 구조

### 프레임워크

- Next.js App Router
- TypeScript
- Tailwind CSS

### 배포/운영

- 로컬에서 개발
- GitHub 원격 저장소에 푸시
- Vercel에서 GitHub 연동 후 자동 배포

### 운영 원칙

- 초기에는 서버, 데이터베이스, 관리자 페이지를 두지 않는다.
- 콘텐츠는 정적 데이터 파일과 이미지 파일 중심으로 관리한다.
- 추후 확장 시 MDX, CMS, 외부 API를 붙일 수 있도록 페이지와 데이터 계층을 분리한다.

## 4. 정보 구조

홈페이지는 하나의 페이지 안에서 다음 순서로 구성한다.

1. 히어로
2. 농원 소개
3. 농원 규모 및 강점
4. 주요 식물 카테고리
5. 농원 둘러보기
6. 식물 관리 팁
7. 농원 소식
8. 오시는 길 및 문의

이 순서는 사용자가 첫 화면에서 브랜드 인상을 받고, 농원의 신뢰 요소를 확인한 뒤, 어떤 식물을 만날 수 있는지 이해하고, 마지막에 방문/연락 행동으로 이어지도록 설계한다.

## 5. UI/브랜드 방향

### 톤 앤 매너

- 깔끔하고 정돈된 브랜드 사이트 인상
- 과하게 장식적이지 않은 농원 이미지
- 친근하지만 가벼워 보이지 않는 문장 톤

### 시각 방향

- 배경은 아이보리, 샌드, 연한 세이지 계열 중심
- 강조색은 잎사귀 느낌의 딥그린 또는 올리브 계열 사용
- 카드, 섹션 구분, 얇은 선, 넉넉한 여백으로 정보 가독성을 확보
- 사진이 없는 영역은 색면, 패턴, 간단한 오버레이로 비주얼 완성도를 유지

### 타이포그래피

- 기본은 가독성 좋은 산세리프 중심
- 제목과 본문 대비를 명확하게 두어 브랜드형 인상을 강화

## 6. 컴포넌트 구조

예상 컴포넌트는 다음과 같다.

- `Header`: 브랜드명, 앵커 메뉴, 모바일 메뉴
- `HeroSection`: 대표 문구, 핵심 설명, 주요 CTA
- `SectionShell`: 공통 섹션 여백과 제목 스타일
- `IntroSection`: 농원 소개와 운영 방향
- `HighlightsSection`: 농원 규모, 위치, 강점 요약
- `PlantCategorySection`: 화분, 묘목, 조경용 식물 소개 카드
- `GallerySection`: 농원/공간/식물 이미지 목록
- `GuideSection`: 초보자용 관리 팁 카드
- `NewsSection`: 농원 소식 카드
- `LocationSection`: 주소, 안내 문구, 길찾기/연락 링크
- `Footer`: 기본 안내 문구와 저작권성 정보

## 7. 데이터 구조

반복 콘텐츠는 `data/` 폴더 아래 TypeScript 상수 파일로 관리한다.

예상 데이터 파일:

- `data/navigation.ts`
- `data/plants.ts`
- `data/guides.ts`
- `data/news.ts`
- `data/gallery.ts`
- `data/contact.ts`

각 데이터는 화면 텍스트, 설명, 링크, 이미지 경로를 포함하며, 페이지 컴포넌트는 해당 데이터를 읽어 렌더링만 담당한다. 이 방식은 다음 장점이 있다.

- 콘텐츠 수정 시 JSX 구조를 건드리지 않아도 된다.
- 실제 운영자가 GitHub에서 텍스트만 교체하기 쉬워진다.
- 나중에 Markdown/MDX 또는 CMS로 이전할 때 마이그레이션 범위를 줄일 수 있다.

## 8. 섹션별 동작

### 히어로

- 농원 핵심 메시지 노출
- `식물 안내 보기`, `오시는 길` 같은 정보 탐색형 CTA 제공
- 첫 화면에서 영천 지역성과 농원 정체성을 명확히 보여준다.

### 농원 소개

- 덕앤플라워의 배경, 방향, 철학을 짧고 신뢰감 있게 전달

### 농원 규모 및 강점

- 창고 300평, 밭 800평 등 실제 보유 자원을 카드나 지표 형태로 제시
- 방문 전 사용자가 농원의 실체감을 느끼도록 한다.

### 주요 식물 카테고리

- 화분, 묘목, 조경용 식물, 계절 식물 등을 카드형으로 요약
- 가격보다는 활용처와 특징 중심으로 설명

### 농원 둘러보기

- 실제 사진이 있으면 우선 사용
- 초기에는 플레이스홀더 또는 대체 비주얼을 넣되, 사진 교체가 쉬운 구조로 구현

### 식물 관리 팁

- 초보자에게 유용한 짧은 정보 제공
- 전문성 전달과 검색 친화성을 함께 노림

### 농원 소식

- 농원 준비 과정, 계절 변화, 입고 소식 등 업데이트 영역
- 초기에는 샘플 데이터로 구성하고, 추후 카드만 추가 가능하게 설계

### 오시는 길 및 문의

- 주소, 지역 설명, 길찾기 링크, 전화/문자/카카오톡 등 외부 연결 중심
- 초기에는 지도 임베드보다 외부 지도 서비스 링크를 우선 사용

## 9. 예외 처리 및 확장 전략

- 실사진이 부족한 상태에서도 완성도 있게 보이도록 이미지 없는 상태를 기본적으로 지원한다.
- 예약/구매 기능은 현재 범위에서 제외하지만, CTA 영역과 데이터 구조는 추후 기능 확장을 방해하지 않도록 중립적으로 작성한다.
- 섹션이 길어질 경우 일부 콘텐츠를 별도 페이지로 분리할 수 있도록 컴포넌트 경계를 명확히 유지한다.

## 10. SEO 및 배포 고려사항

- 페이지 메타 정보에 `덕앤플라워`, `영천`, `화분`, `묘목`, `농원` 관련 키워드를 반영한다.
- 단일 페이지이더라도 섹션 제목과 카피를 명확히 구성해 검색엔진이 페이지 주제를 이해하기 쉽게 한다.
- GitHub 저장소에 푸시한 뒤 Vercel 프로젝트와 연결해 자동 배포가 가능하도록 일반적인 Next.js 구조를 유지한다.
- 공개 후 실제 도메인을 연결할 경우를 고려해 기본 메타데이터와 오픈그래프 구조를 준비한다.

## 11. 테스트 범위

- 데스크톱과 모바일에서 섹션 간 가독성 확인
- 앵커 네비게이션 동작 확인
- CTA 링크 동작 확인
- 데이터 파일 수정 시 각 섹션이 정상 렌더링되는지 확인
- Vercel 배포 기준으로 빌드 가능한 정적 구조 유지

## 12. 구현 범위 요약

이번 구현 범위는 다음에 한정한다.

- Next.js 기반 프로젝트 초기화
- 단일 페이지 랜딩 구조 구현
- 데이터 분리형 섹션 렌더링 구현
- 기본 반응형 스타일 적용
- GitHub/Vercel 배포를 고려한 프로젝트 구조 정리

이번 범위에 포함하지 않는 항목은 다음과 같다.

- 예약 기능
- 온라인 구매/결제
- 회원가입/로그인
- 관리자 페이지
- 데이터베이스/백엔드 API

## 13. 디자인 리디자인 보강안

### 목표

초기 정보형 랜딩 구조는 유지하되, 시각 언어는 `Garsons` 스타일에 가깝게 재구성한다. 단, 단순 복제가 아니라 한국어 콘텐츠 길이, 덕앤플라워의 농원 정보 구조, 실제 운영 목적에 맞게 현지화한다.

### 레퍼런스 적용 원칙

- 레이아웃은 Garsons처럼 넓은 히어로와 여백 중심의 브랜드형 구조를 따른다.
- 색감은 크림, 아이보리, 올리브, 딥그린, 웜 브라운 계열을 중심으로 정리한다.
- 섹션은 정보 나열형이 아니라 `브랜드 소개 → 주요 카테고리 → 공간/이미지 → 관리/소식 → 방문 유도` 흐름으로 보이게 재구성한다.
- 카피는 한국어로 유지하되, 문장 길이는 짧고 신뢰감 있게 다듬는다.
- CTA는 판매 전환보다 `더 알아보기`, `식물 안내 보기`, `오시는 길` 같은 정보 탐색형 구조를 유지한다.

### 레이아웃 방향

- 헤더는 얇고 정제된 상단 바 형태로 유지
- 히어로는 대형 이미지 또는 대형 이미지형 비주얼 블록 중심
- 중간 섹션은 카드형 카테고리와 넓은 이미지 블록을 교차 배치
- 하단 섹션은 관리 팁, 소식, 방문 안내를 균형감 있게 정렬
- 모바일에서도 카드와 이미지가 무너지지 않도록 1열 기준으로 자연스럽게 정리

### 카피 방향

- 제목은 더 짧고 상업적이지 않게 정리
- 설명문은 문단형보다는 브랜드형 보조 설명으로 압축
- 영어 레이블은 최소화하고, 필요 시에도 보조적 역할만 하도록 조정

### 시각적 디테일

- 버튼은 둥근 형태와 묵직한 색 대비 사용
- 카드 테두리는 약하게, 배경은 질감이 느껴지는 밝은 톤으로 유지
- 플레이스홀더 이미지도 단순 도형보다 브랜드 무드가 느껴지게 정리
- 실제 농원 사진으로 교체했을 때 가장 자연스럽게 연결되도록 이미지 비율과 프레임을 먼저 맞춘다.
