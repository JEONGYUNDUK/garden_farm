# 덕앤플라워 홈페이지 디자인 스펙

## 1. 목표

덕앤플라워 홈페이지는 경북 영천의 농원 정보를 신뢰감 있게 전달하고, 사용자가 농원을 직접 방문해 보고 싶다는 인상을 갖게 만드는 정보 제공형 브랜드 사이트로 구축한다. 초기 버전은 예약, 구매, 회원가입 없이 빠르게 공개 가능한 정적 사이트를 목표로 하며, 로컬 작업 후 GitHub에 푸시하고 Vercel로 배포하는 운영 방식을 전제로 한다.

## 2. 제품 방향

- 사이트 형태는 다중 페이지가 아닌 단일 페이지 랜딩 구조로 한다.
- 상단 네비게이션은 각 섹션으로 이동하는 앵커 링크 방식으로 구성한다.
- 화면 분위기는 깔끔한 브랜드형을 기본으로 하되, 농원의 자연스러움은 색상과 이미지 톤으로 보완한다.
- 반복 콘텐츠는 컴포넌트 내부 하드코딩 대신 별도 데이터 파일로 분리해 유지보수를 쉽게 한다.
- 초기 공개 후 실제 농원 사진, 소식, 식물 정보가 늘어나더라도 구조 변경 없이 데이터만 추가할 수 있도록 설계한다.

## 3. 기술 구조

### 프레임워크

- Next.js App Router
- TypeScript
- Tailwind CSS

### 배포/운영

- 로컬에서 개발
- GitHub 원격 저장소에 푸시
- Vercel에서 GitHub 연동 후 자동 배포

### 운영 원칙

- 초기에는 서버, 데이터베이스, 관리자 페이지를 두지 않는다.
- 콘텐츠는 정적 데이터 파일과 이미지 파일 중심으로 관리한다.
- 추후 확장 시 MDX, CMS, 외부 API를 붙일 수 있도록 페이지와 데이터 계층을 분리한다.

## 4. 정보 구조

홈페이지는 하나의 페이지 안에서 다음 순서로 구성한다.

1. 히어로
2. 농원 소개
3. 농원 규모 및 강점
4. 주요 식물 카테고리
5. 농원 둘러보기
6. 식물 관리 팁
7. 농원 소식
8. 오시는 길 및 문의

이 순서는 사용자가 첫 화면에서 브랜드 인상을 받고, 농원의 신뢰 요소를 확인한 뒤, 어떤 식물을 만날 수 있는지 이해하고, 마지막에 방문/연락 행동으로 이어지도록 설계한다.

## 5. UI/브랜드 방향

### 톤 앤 매너

- 깔끔하고 정돈된 브랜드 사이트 인상
- 과하게 장식적이지 않은 농원 이미지
- 친근하지만 가벼워 보이지 않는 문장 톤

### 시각 방향

- 배경은 아이보리, 샌드, 연한 세이지 계열 중심
- 강조색은 잎사귀 느낌의 딥그린 또는 올리브 계열 사용
- 카드, 섹션 구분, 얇은 선, 넉넉한 여백으로 정보 가독성을 확보
- 사진이 없는 영역은 색면, 패턴, 간단한 오버레이로 비주얼 완성도를 유지

### 타이포그래피

- 기본은 가독성 좋은 산세리프 중심
- 제목과 본문 대비를 명확하게 두어 브랜드형 인상을 강화

## 6. 컴포넌트 구조

예상 컴포넌트는 다음과 같다.

- `Header`: 브랜드명, 앵커 메뉴, 모바일 메뉴
- `HeroSection`: 대표 문구, 핵심 설명, 주요 CTA
- `SectionShell`: 공통 섹션 여백과 제목 스타일
- `IntroSection`: 농원 소개와 운영 방향
- `HighlightsSection`: 농원 규모, 위치, 강점 요약
- `PlantCategorySection`: 화분, 묘목, 조경용 식물 소개 카드
- `GallerySection`: 농원/공간/식물 이미지 목록
- `GuideSection`: 초보자용 관리 팁 카드
- `NewsSection`: 농원 소식 카드
- `LocationSection`: 주소, 안내 문구, 길찾기/연락 링크
- `Footer`: 기본 안내 문구와 저작권성 정보

## 7. 데이터 구조

반복 콘텐츠는 `data/` 폴더 아래 TypeScript 상수 파일로 관리한다.

예상 데이터 파일:

- `data/navigation.ts`
- `data/plants.ts`
- `data/guides.ts`
- `data/news.ts`
- `data/gallery.ts`
- `data/contact.ts`

각 데이터는 화면 텍스트, 설명, 링크, 이미지 경로를 포함하며, 페이지 컴포넌트는 해당 데이터를 읽어 렌더링만 담당한다. 이 방식은 다음 장점이 있다.

- 콘텐츠 수정 시 JSX 구조를 건드리지 않아도 된다.
- 실제 운영자가 GitHub에서 텍스트만 교체하기 쉬워진다.
- 나중에 Markdown/MDX 또는 CMS로 이전할 때 마이그레이션 범위를 줄일 수 있다.

## 8. 섹션별 동작

### 히어로

- 농원 핵심 메시지 노출
- `식물 안내 보기`, `오시는 길` 같은 정보 탐색형 CTA 제공
- 첫 화면에서 영천 지역성과 농원 정체성을 명확히 보여준다.

### 농원 소개

- 덕앤플라워의 배경, 방향, 철학을 짧고 신뢰감 있게 전달

### 농원 규모 및 강점

- 창고 300평, 밭 800평 등 실제 보유 자원을 카드나 지표 형태로 제시
- 방문 전 사용자가 농원의 실체감을 느끼도록 한다.

### 주요 식물 카테고리

- 화분, 묘목, 조경용 식물, 계절 식물 등을 카드형으로 요약
- 가격보다는 활용처와 특징 중심으로 설명

### 농원 둘러보기

- 실제 사진이 있으면 우선 사용
- 초기에는 플레이스홀더 또는 대체 비주얼을 넣되, 사진 교체가 쉬운 구조로 구현

### 식물 관리 팁

- 초보자에게 유용한 짧은 정보 제공
- 전문성 전달과 검색 친화성을 함께 노림

### 농원 소식

- 농원 준비 과정, 계절 변화, 입고 소식 등 업데이트 영역
- 초기에는 샘플 데이터로 구성하고, 추후 카드만 추가 가능하게 설계

### 오시는 길 및 문의

- 주소, 지역 설명, 길찾기 링크, 전화/문자/카카오톡 등 외부 연결 중심
- 초기에는 지도 임베드보다 외부 지도 서비스 링크를 우선 사용

## 9. 예외 처리 및 확장 전략

- 실사진이 부족한 상태에서도 완성도 있게 보이도록 이미지 없는 상태를 기본적으로 지원한다.
- 예약/구매 기능은 현재 범위에서 제외하지만, CTA 영역과 데이터 구조는 추후 기능 확장을 방해하지 않도록 중립적으로 작성한다.
- 섹션이 길어질 경우 일부 콘텐츠를 별도 페이지로 분리할 수 있도록 컴포넌트 경계를 명확히 유지한다.

## 10. SEO 및 배포 고려사항

- 페이지 메타 정보에 `덕앤플라워`, `영천`, `화분`, `묘목`, `농원` 관련 키워드를 반영한다.
- 단일 페이지이더라도 섹션 제목과 카피를 명확히 구성해 검색엔진이 페이지 주제를 이해하기 쉽게 한다.
- GitHub 저장소에 푸시한 뒤 Vercel 프로젝트와 연결해 자동 배포가 가능하도록 일반적인 Next.js 구조를 유지한다.
- 공개 후 실제 도메인을 연결할 경우를 고려해 기본 메타데이터와 오픈그래프 구조를 준비한다.

## 11. 테스트 범위

- 데스크톱과 모바일에서 섹션 간 가독성 확인
- 앵커 네비게이션 동작 확인
- CTA 링크 동작 확인
- 데이터 파일 수정 시 각 섹션이 정상 렌더링되는지 확인
- Vercel 배포 기준으로 빌드 가능한 정적 구조 유지

## 12. 구현 범위 요약

이번 구현 범위는 다음에 한정한다.

- Next.js 기반 프로젝트 초기화
- 단일 페이지 랜딩 구조 구현
- 데이터 분리형 섹션 렌더링 구현
- 기본 반응형 스타일 적용
- GitHub/Vercel 배포를 고려한 프로젝트 구조 정리

이번 범위에 포함하지 않는 항목은 다음과 같다.

- 예약 기능
- 온라인 구매/결제
- 회원가입/로그인
- 관리자 페이지
- 데이터베이스/백엔드 API

## 13. 2차 리디자인 보강안

### 목표

현재 단일 페이지 구조와 데이터 분리 방식은 유지하되, 화면 인상은 `고급스럽고 차분한 영국식 가든센터`에 가깝게 전면 재정리한다. 단, Garsons의 로고, 문구, 브랜드 자산은 복사하지 않고, 덕앤플라워만의 정보형 농원 홈페이지로 보이게 현지화한다.

### 리디자인 핵심 원칙

- 목적은 판매 전환이 아니라 `농원 정보 제공을 통해 방문 관심을 높이는 것`에 둔다.
- 쇼핑몰처럼 보이는 배지, 가격 강조, 강한 프로모션 표현은 사용하지 않는다.
- 사진, 여백, 간결한 카피를 중심으로 신뢰감 있는 농원 소개 인상을 만든다.
- 한국어 문장 길이에 맞게 카피 밀도를 줄이고, 한 화면에서 읽히는 정보량을 절제한다.

### 컬러 시스템

- 기본 배경: `#FFFFFF`
- 보조 배경: `#F4F1EC`, `#F5F5F0`
- 메인 포인트: `#1F3D2B`, `#23412F`
- 서브 포인트: `#6F8F72`, `#8FA58A`
- 기본 텍스트: `#2B2B2B`
- 보조 텍스트: `#666666`
- 푸터 배경: `#242424` 또는 `#1F2420`
- 버튼/라인 보조: `#FFFFFF`, `#D8D5CF`, `#1F3D2B`

적용 원칙은 `화이트 기반 + 짙은 차콜/다크그린 포인트 + 세이지/베이지 보조색`이다. 초록은 배경 전체를 덮는 용도가 아니라 포인트와 이미지 무드 보강에만 사용한다.

### 레이아웃 방향

- 상단 헤더는 흰 배경 위에 넓은 여백과 얇은 경계선으로 정리한다.
- 헤더는 좌측 `menu`, 중앙 `덕앤플라워` 텍스트 로고, 우측 `전화/카카오톡/지도/검색` 링크 구조로 재배치한다.
- 메인 첫 화면은 큰 이미지 중심 히어로로 바꾸고, 이미지 위에 어두운 오버레이와 핵심 카피를 올린다.
- 히어로 바로 아래에는 가로형 이미지 카드 3개를 배치해 `농원 소개`, `화분·묘목 안내`, `오시는 길`로 진입을 유도한다.
- 이후 섹션은 `소개 → 농원 둘러보기 → 화분·묘목 안내 → 방문 안내 → 문의 안내` 흐름이 더 분명하게 읽히도록 정리한다.

### 헤더 디자인 규칙

- 중앙 브랜드 표기는 `덕앤플라워`
- 브랜드 보조 문구는 `영천 화분·묘목 농원`
- 좌측은 햄버거 버튼 또는 `menu` 텍스트 버튼
- 우측은 실제 기능이 있는 범위에서 전화, 카카오톡, 지도, 검색 형태의 간단한 링크/아이콘
- 헤더는 시선을 과하게 끌지 않고, 브랜드의 정돈된 첫인상 역할에 집중한다.

### 히어로 디자인 규칙

- 큰 농원 사진 또는 식물 사진 1장을 메인으로 사용
- 이미지 위에 짙은 오버레이를 얇게 깔아 텍스트 가독성을 높인다.
- 히어로 카피는 다음 문구를 기본으로 한다.
  - 메인 제목: `덕앤플라워`
  - 서브 문구: `경북 영천에서 만나는 화분·묘목 농원`
  - 설명 문구: `800평 밭과 300평 창고를 기반으로 다양한 화분과 묘목 정보를 소개합니다.`
- 버튼은 `농원 둘러보기`, `오시는 길 보기` 2개를 기본으로 하며, 흰 테두리 버튼 또는 다크그린 버튼 조합을 사용한다.

### 메인 카드 영역 규칙

- 히어로 아래에 가로형 이미지 카드 3개를 배치한다.
- 카드 주제는 다음 3개를 기본값으로 한다.
  - `농원 소개`
  - `화분·묘목 안내`
  - `오시는 길`
- 각 카드는 이미지 배경 위에 흰색 테두리 버튼형 텍스트를 얹는다.
- 마우스 오버 시 이미지가 살짝 확대되거나 어두워지는 정도의 절제된 인터랙션만 사용한다.
- 카드 사이 간격은 좁지 않게 유지해 여백감이 느껴지게 한다.

### 섹션 구조 규칙

#### 덕앤플라워 소개

- 위치: `경북 영천 화산 효정리`
- 규모: `800평 밭`, `300평 창고`
- 역할: `화분, 묘목, 조경용 식물 정보를 제공하는 농원`

#### 농원 둘러보기

- 밭, 창고, 식물, 작업 공간을 보여주는 갤러리 카드 그리드
- 실제 사진이 없으면 플레이스홀더를 유지하되, 나중에 파일 교체만으로 반영되게 구성

#### 화분·묘목 안내

- `화분`, `묘목`, `조경수`, `계절 식물` 4개 카테고리 카드 중심
- 가격보다 용도, 분위기, 배치 공간 중심 설명

#### 방문 안내

- 주소
- 주차 가능 여부
- 방문 전 전화 문의 권장
- 네이버지도 또는 카카오맵 외부 링크 버튼

#### 문의 안내

- 전화 문의
- 문자 문의
- 카카오톡 문의
- 지도 보기

### 푸터 방향

- 첨부 레퍼런스처럼 어두운 배경을 사용한다.
- 왼쪽에는 `덕앤플라워` 로고형 텍스트
- 가운데에는 주소와 연락처
- 오른쪽에는 주요 섹션 링크 또는 SNS/외부 링크
- 하단에는 `© Deok & Flower. All rights reserved.` 형태의 저작권 문구를 둔다.

### 타이포그래피 방향

- 본문은 `Pretendard`, `Noto Sans KR`, 시스템 산세리프 계열 우선
- 제목은 더 굵고 간결하게
- 로고형 텍스트는 본문보다 감성적인 serif 또는 editorial 계열을 보조적으로 사용
- 줄간격은 넉넉하게 유지해 한글 가독성을 높인다.

### 모바일 반응형

- 메인 3카드는 모바일에서 1열 세로 스택
- 메뉴는 햄버거 메뉴로 정리
- 이미지와 버튼 크기를 유지해 작은 화면에서도 브랜드 인상이 약해지지 않게 한다.
- 필요하면 하단 고정 `전화 / 카카오톡 / 길찾기` CTA 바를 둔다.

### 구현 제약

- 현재 Next.js, React, Tailwind CSS 구조를 유지한다.
- 불필요한 라이브러리는 추가하지 않는다.
- 예약, 구매, 장바구니, 로그인, 회원가입 기능은 추가하지 않는다.
- 정적 홈페이지 구조를 유지한다.
- SEO용 title, description은 유지 또는 개선한다.
- 이미지 파일은 추후 실제 농원 사진으로 교체하기 쉽게 `public/images` 기준으로 관리한다.
